**修改了三个文件，旧代码完整保留：**

### [robojudo/controller/utils/joystick.py](robojudo/controller/utils/joystick.py)
在 `JoystickThread`（pygame）之后、`unitreeRemoteController` 之前，新增了 `PyUSBJoystickThread` 类：
- `__init__` 时就搜索 USB 设备，找不到则抛 `RuntimeError`（方便上层 fallback）
- 支持 **Xbox 360 wired** 和 **Xbox One wired**（含 Elite/S/Series X·S），按协议类型分别解析原始 USB 报文
- 生成与 `JoystickThread` **完全相同格式**的 `state_queue`（axes）和 `event_queue`（button 事件），上层代码零改动
- 自动 detach 残留的 kernel driver（如 xpad 部分加载的情况）

### [robojudo/controller/ctrl_cfgs.py](robojudo/controller/ctrl_cfgs.py)
`JoystickCtrlCfg` 新增字段：
```python
use_pyusb: bool = True
```
默认开启 pyusb 模式；设为 `False` 可回退到 pygame 旧路径。

### [robojudo/controller/joystick_ctrl.py](robojudo/controller/joystick_ctrl.py)
`JoystickCtrl.__init__` 中增加后端选择逻辑：
- `use_pyusb=True`（默认）：优先用 `PyUSBJoystickThread`；若设备未接入或 pyusb 未安装，**自动 fallback 到 `JoystickThread`（pygame）** 并打印警告

```bash
sudo udevadm control --reload && sudo udevadm trigger
```

**第二步：按情况处理**

**情况 A — 北通手柄已在内置列表里（PID 是 5500/5501/5506）**  
无需改配置，直接运行即可。

**情况 B — PID 不在列表里**  
在你的配置文件里给 `JoystickCtrlCfg` 加两行，把 `0x20bc` 和 `0x????` 换成 `lsusb` 看到的实际值：
```python
JoystickCtrlCfg(
    pyusb_vid=0x20bc,   # 北通 VID（示例）
    pyusb_pid=0x5500,   # 你的实际 PID
    ...
)
```

---

**关于 XInput 协议**：北通手柄的 XInput 模式与 Xbox 360 使用完全相同的 USB 报文格式（20 字节），所以只要 VID/PID 匹配上，数据解析这块不需要任何改动。
